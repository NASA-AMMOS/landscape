# CLAUDE.md — Open Mission Software Landscape

This file gives Claude Code persistent context for this repo. Loaded at
the start of every session.

## Why this repo exists

This is the data and tooling for an interactive **open-source mission
software landscape**, generated by [CNCF landscape2](https://github.com/cncf/landscape2)
and published to GitHub Pages. It catalogs OSS used in space mission
design, ground systems, mission operations, and science data processing.
Maintained by NASA-AMMOS; submissions welcome from any NASA center,
partner org, university, commercial space company, or community project
that meets the inclusion criteria.

## What's where

- `landscape.yml` — the canonical data file. One entry per item, nested
  under `categories[].subcategories[].items[]`. Items inside a
  subcategory are kept alphabetical by `name`.
- `settings.yml` — landscape2 colors, groups, facets, category ordering.
- `guide.yml` — per-category descriptions for the Guide tab.
- `logos/` — SVG logos. `placeholder.svg` is the temporary fallback for
  items without a real logo yet.
- `tools/transform_code_json.py` — one-shot helper that proposes items
  from `nasa/Open-Source-Catalog` for triage. Not run on a schedule.
- `tools/validate_local.sh` — runs the same checks CI runs, locally.
- `docs/annotation-schema.md` — the `submitter.*` and `nasa.*` annotation
  conventions. Read this before adding items.
- `docs/category-taxonomy.md` — what category goes where.
- `docs/governance.md` — review process, scope, partner submissions.
- `.github/workflows/validate.yml` — PR schema + annotation checks.
- `.github/workflows/build-and-deploy.yml` — main-branch build + Pages deploy.
- `CODEOWNERS` — `@NASA-AMMOS/landscape-maintainers` reviews everything.

## How to work in this repo

### Inclusion criteria (all four must hold)

1. License is OSI-approved or NASA Open Source Agreement (NASA-1.3).
2. Software is mission-related (mission design, ground systems, flight
   software, mission ops, science data, or directly supporting standards).
3. Public code repository exists (GitHub strongly preferred).
4. Reasonably maintained or historically significant.

Out of scope: closed-source, government-source-only, ITAR/EAR-controlled,
or non-mission software.

### Adding an item

1. Pick the category and subcategory in `docs/category-taxonomy.md`.
2. Edit `landscape.yml`. Insert alphabetically by `name` in the
   subcategory's `items:` list.
3. Required fields: `name`, `homepage_url`, `logo`, `description`,
   `repo_url`, `license`.
4. Required annotations under `extra.annotations`:
   - `submitter.org` (NASA center code or partner org name)
   - `submitter.type` (one of: `nasa-center`, `commercial-partner`,
     `academic`, `agency-partner`, `community`, `consortium`)
5. For NASA-maintained items, also set `nasa.center` and
   `nasa.distribution: open-source`. Optional NASA fields:
   `nasa.ammos_element`, `nasa.missions`, `nasa.standards`,
   `nasa.sponsor_program`.
6. If no real SVG logo exists yet, reference `placeholder.svg`. File a
   follow-up issue tagged `logo-needed`.

### Commands

```bash
# Validate everything (matches CI). Requires Docker.
./tools/validate_local.sh

# Build the static site locally (requires landscape2 CLI)
landscape2 build --data-file landscape.yml --settings-file settings.yml \
  --guide-file guide.yml --logos-path logos --output-dir build

# Serve it
landscape2 serve --landscape-dir build

# Run the one-shot code.json transformer (initial seed / bulk add).
# Python here is managed by uv (PEP 668 blocks plain `pip install`),
# so use one of:
uv run --with-requirements tools/requirements.txt \
  python3 tools/transform_code_json.py --output tools/_proposed/

# ...or set up a project venv once, then activate it:
#   uv venv && uv pip install -r tools/requirements.txt
#   source .venv/bin/activate
#   python3 tools/transform_code_json.py --output tools/_proposed/
```

### Conventions

- **Single file:** all data lives in `landscape.yml`. There is no
  `data.yml`, no `categories/` directory, no merge step.
- **Alphabetical ordering** within each subcategory.
- **Don't reproduce the existing AMMOS Catalog.** Closed items are out
  of scope. Items that happen to be both in the Catalog and open-source
  qualify on the open-source side only.
- **`tools/_proposed/` is gitignored.** Outputs from the transformer
  are for review, not for commit. Approved items get hand-copied into
  `landscape.yml`.
- **Annotations are strings.** Lists are comma-separated strings, not
  YAML lists. The free-form annotation map is `extra.annotations` —
  keys must be strings, values must be strings.

### When fixing CI failures

Validate failures usually fall into three buckets:

1. **Schema** — landscape2 schema rejected something. Run
   `./tools/validate_local.sh` to see the error inline.
2. **Logo** — referenced an SVG that doesn't exist in `logos/`. Either
   add it or reference `placeholder.svg`.
3. **Annotations** — missing `submitter.org` or `submitter.type`, or
   `nasa.distribution` is set to anything other than `open-source`.

Before opening a PR, run `./tools/validate_local.sh` locally.
